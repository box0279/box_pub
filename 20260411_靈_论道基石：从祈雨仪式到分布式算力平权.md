## 🌩️ 序言：为何我们需要「灵动」的网络？

中东甲骨文机房的轰然倒塌，是中心化巴别塔在物理冲突面前脆弱性的明证。当算力被囚禁在巨型建筑内，它便有了被「斩首」的风险。

而我们正在构建的「论道台」，追求的是一种像生命体一样的**「群体智能」**。它不应该有中心，它的每一条物理触角（Edge Node）都应该是自治且强韧的。这次对 MT3000 路由器的深度重构，便是我为这个系统埋下的第一颗「靈」之种子。

## 🏮 哲思：何为「靈」？

在这次重构中，我突然对「靈」这个字有了纵向贯通的体悟：

* **上覆「雨」**：那是天意的垂青，是积蓄在云端的分布式算力势能，等待着被降下。
* **中列「三口」**：那是我们建立的 API 接口与 ZTNA 隧道，是承接甘露的端口，也是众生（节点）共鸣的呼唤。
* **下驻「巫」**：立于天地之间，以代码为舞。开发者不再枯坐，而是通过主动的介入，让系统从「静止」转向「流转」。

**秘密之所以成为秘密，是因为它在等待合适的频率。** 当这套低成本、抗断电、能自愈的方案跑通时，频率对上了，「靈」便显现了。

## 🛠️ 技术 SOP：打造你的“算力神经元”

如果你也想加入这场分布式算力的共建，请按照以下步骤，将一台普通的 GL-MT3000 转化成具备“自愈能力”的网关。

### 1. 固件底座
* **固件**：`immortalwrt-23.05.3-mediatek-filogic-glinet_gl-mt3000-squashfs-sysupgrade.bin`
* **操作**：本地升级，不保留配置，重塑纯净肉身。

### 2. 配置还原（注入灵魂）
* **快照**：`backup-ImmortalWrt-2026-04-12.tar.gz`
* **操作**：在 `系统 -> 备份与升级` 中上传。它包含了：
    * **DNS 净化**：无视运营商劫持，强制 8.8.8.8。
    * **持久化**：将 Tailscale 状态从 tmpfs 迁移至 Flash，实现“断电不失忆”。
    * **防火墙**：自动打通隧道与 WAN 区的 NAT 映射。

### 3. 子网互联（开启触角）
在 SSH 终端执行终极并网指令，实现跨地域局域网互通：
```bash
tailscale up --exit-node=100.87.20.49 --exit-node-allow-lan-access --accept-dns=false --advertise-routes=192.168.8.0/24 --accept-routes
```

### 4. 固化自愈脚本
修改 `/etc/rc.local`，确保断电重启后 30 秒内自动回位：
```bash
sed -i 's/tailscale up.*/tailscale up --exit-node=100.87.20.49 --exit-node-allow-lan-access --accept-dns=false --advertise-routes=192.168.8.0\/24 --accept-routes \&/' /etc/rc.local
```
